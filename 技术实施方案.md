# 校园二手物品交易平台技术实施方案

## 一、技术选型

### 1.1 前端技术
- **模板引擎**：EJS（Embedded JavaScript）—— 轻量级、服务端渲染
- **CSS框架**：Bootstrap 5 —— 响应式设计、移动端适配
- **页面交互**：原生JavaScript + jQuery（简化DOM操作）
- **页面跳转**：传统超链接（`<a>`标签）+ 表单提交

### 1.2 后端技术
- **框架**：Express.js —— Node.js轻量级Web框架
- **数据库**：MySQL 8.0 —— 关系型数据库
- **会话管理**：express-session —— 基于Cookie的会话管理
- **文件上传**：multer —— 处理商品图片上传
- **密码加密**：bcryptjs —— 用户密码安全存储

### 1.3 部署方案
- **托管平台**：腾讯云CloudBase Web托管
- **Node.js版本**：16.x / 18.x

## 三、数据库设计

### 3.1 用户表（users）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 用户ID |
| phone | VARCHAR(11) UNIQUE | 手机号（登录账号） |
| password | VARCHAR(255) | 加密后的密码 |
| nickname | VARCHAR(50) | 用户昵称 |
| role | ENUM('user', 'admin') | 用户角色 |
| created_at | DATETIME | 注册时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 商品表（products）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 商品ID |
| user_id | INT FOREIGN KEY | 卖家ID |
| category | VARCHAR(50) | 分类 |
| title | VARCHAR(50) | 标题 |
| description | TEXT | 描述 |
| price | DECIMAL(10,2) | 价格 |
| images | JSON | 图片路径数组 |
| status | ENUM('onsale', 'offsale', 'sold') | 状态 |
| created_at | DATETIME | 发布时间 |
| updated_at | DATETIME | 更新时间 |

### 3.3 订单表（orders）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 订单ID |
| product_id | INT FOREIGN KEY | 商品ID |
| buyer_id | INT FOREIGN KEY | 买家ID |
| seller_id | INT FOREIGN KEY | 卖家ID |
| receiver_name | VARCHAR(50) | 收货人 |
| receiver_phone | VARCHAR(11) | 联系电话 |
| receiver_address | VARCHAR(255) | 收货地址 |
| total_price | DECIMAL(10,2) | 订单金额 |
| status | ENUM('pending', 'paid', 'shipped', 'completed', 'cancelled') | 订单状态 |
| created_at | DATETIME | 下单时间 |
| updated_at | DATETIME | 更新时间 |

## 四、核心功能模块

### 4.1 用户模块
- **注册**：手机号 + 密码注册，密码bcrypt加密
- **登录**：手机号+密码登录，session会话管理
- **个人中心**：查看/编辑个人信息、查看发布的商品

### 4.2 商品模块
- **发布商品**：表单提交，图片上传（multer），最多6张
- **商品列表**：支持排序（最新/价格升序/价格降序）
- **商品详情**：图片轮播，卖家信息展示
- **搜索筛选**：关键词搜索，分类/价格区间筛选

### 4.3 交易模块
- **立即购买**：填写收货信息，生成订单
- **订单管理**：查看订单记录

### 4.4 管理后台
- **用户管理**：查看用户列表
- **商品管理**：强制删除商品

## 五、部署步骤

### 5.1 本地开发
```bash
# 安装依赖
npm install

# 初始化数据库
mysql -u root -p < database/schema.sql

# 启动开发服务器
npm run dev
# 或
node app.js
```

### 5.2 CloudBase部署
```bash
# 安装CloudBase CLI
npm install -g @cloudbase/cli

# 登录
tcb login

# 部署Web应用
tcb web deploy
```

## 七、技术优势

1. **快速交付**：服务端渲染减少前后端联调时间
2. **维护简单**：传统MVC架构，代码结构清晰
3. **成本低**：CloudBase免费额度足够个人项目使用
4. **响应式**：Bootstrap确保多端适配
5. **敏捷开发**：每个阶段可独立测试交付
